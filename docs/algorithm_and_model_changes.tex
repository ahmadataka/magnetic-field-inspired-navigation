\subsection*{What Was Implemented}

The current cleanroom reference includes:

\begin{itemize}[leftmargin=*]
  \item 2D double-integrator dynamics
  \item 2D circular obstacle abstraction
  \item goal-relaxation gain logic inspired by the ROS implementation
  \item magnetic-field-inspired obstacle avoidance based on the ROS \texttt{field=2} branch
  \item additive total command
\end{itemize}

\begin{align}
a &= a_{\text{goal}} + a_{\text{obs}} \\
v_{k+1} &= v_k + a_k \Delta t \\
p_{k+1} &= p_k + v_{k+1}\Delta t
\end{align}

\subsection*{Goal Relaxation Implemented}

Let
\begin{itemize}[leftmargin=*]
  \item $p \in \mathbb{R}^2$ be robot position
  \item $v \in \mathbb{R}^2$ be robot velocity
  \item $g \in \mathbb{R}^2$ be goal position
  \item $r_g = g - p$ be the goal vector
  \item $r_o$ be the closest obstacle vector
  \item $d_g = \lVert r_g \rVert$
  \item $d_o = \lVert r_o \rVert$
\end{itemize}

The relaxation scale used in the cleanroom implementation is

\begin{equation}
k_{\text{relax}}
=
\left(1 - e^{-d_o/1.5}\right)
\cdot
w_{\angle}
\cdot
w_{\text{prog}}
\cdot
w_{\text{exit}}
\end{equation}

where

\begin{equation}
w_{\angle}
=
1 - \cos \theta
=
1 - \frac{r_g^\top r_o}{\lVert r_g \rVert \lVert r_o \rVert}
\end{equation}

\begin{equation}
w_{\text{prog}}
=
\begin{cases}
1, & d_g \le d_{g,0} \\
\exp\left(-\dfrac{d_g - d_{g,0}}{0.1}\right), & d_g > d_{g,0}
\end{cases}
\end{equation}

and

\begin{equation}
w_{\text{exit}}
=
\exp\left(-\dfrac{d_g - d_{g,\min}}{0.1}\right)
\end{equation}

with:

\begin{itemize}[leftmargin=*]
  \item $d_{g,0}$: initial distance to the goal
  \item $d_{g,\min}$: smallest distance to the goal reached so far
\end{itemize}

This follows the structure of the ROS code around the \texttt{goal\_relax == 1} block.

\subsection*{Goal Controller Implemented}

For the far field, the current cleanroom implementation uses a simplified
speed-and-turn controller inspired by the ROS geometric block:

\begin{equation}
a_{\text{goal}} = a_{\text{attr}} + a_{\text{turn}}
\end{equation}

\begin{equation}
a_{\text{attr}}
=
-\left(k_p\,k_{\text{relax}}\right)
\left(\lVert v \rVert - v_{\max}\right)\hat{s}
\end{equation}

where $\hat{s}$ is:

\begin{itemize}[leftmargin=*]
  \item the normalized velocity direction if $\lVert v \rVert > 0$
  \item otherwise the normalized goal direction
\end{itemize}

The turning term is

\begin{equation}
a_{\text{turn}}
=
k_{\omega}\,k_{\text{relax}}\,\lVert v \rVert\,\hat{n}\,(\hat{n}^\top \hat{g})
\end{equation}

where:

\begin{itemize}[leftmargin=*]
  \item $\hat{g} = r_g / \lVert r_g \rVert$
  \item $\hat{n}$ is the left-perpendicular of the current heading
\end{itemize}

Near the goal, the controller switches to a PD-like attractive form:

\begin{equation}
a_{\text{goal}}
=
k_{p,\text{goal}} r_g - k_d v
\end{equation}

\subsection*{Magnetic-Field-Inspired Obstacle Term Implemented}

The cleanroom port uses the \texttt{field == 2} structure from the old code.

Let
\begin{itemize}[leftmargin=*]
  \item $r_o$ be the closest vector from robot to obstacle surface
  \item $d = \lVert r_o \rVert$
  \item $\hat{l}_a = v / \lVert v \rVert$ be the normalized robot motion direction
  \item $r_{\text{from obs}} = -r_o$
  \item $\hat{r} = r_{\text{from obs}} / \lVert r_{\text{from obs}} \rVert$
\end{itemize}

The obstacle current direction is the tangential projection of the robot motion:

\begin{equation}
\hat{t}
=
\hat{l}_a - (\hat{l}_a^\top \hat{r})\hat{r}
\end{equation}

If this becomes degenerate, the implementation falls back to a perpendicular
direction.

The 2D magnetic field surrogate is modeled as

\begin{equation}
B \propto \frac{\lVert v \rVert}{d}\,\hat{t}_{\perp}
\end{equation}

and the magnetic avoidance term is

\begin{equation}
F_{\text{mag}}
=
k_m \hat{l}_{a,\perp}(\hat{l}_{a,\perp}^\top B)
\end{equation}

This is a 2D analogue of the nested cross-product structure in the ROS code:

\begin{align}
B &= [\hat{t}]_{\times} v / d \\
F &= k_m [\hat{l}_a]_{\times} B
\end{align}

\subsection*{Added Repulsive Correction Implemented}

The cleanroom implementation also keeps the old additional repulsive term near
the obstacle:

\begin{equation}
F_{\text{rep}}
=
-k_r \left(\frac{1}{d} - \frac{1}{d_b}\right)\frac{r_o}{d}
\quad \text{for } d \le d_b
\end{equation}

Then it is projected to remove the component along the motion direction:

\begin{equation}
F_{\text{rep}}^{\perp}
=
F_{\text{rep}} - (F_{\text{rep}}^\top \hat{l}_a)\hat{l}_a
\end{equation}

The final obstacle term is:

\begin{equation}
a_{\text{obs}} = F_{\text{mag}} + F_{\text{rep}}^{\perp}
\end{equation}

\subsection*{Major Simplifications Relative To The Old ROS Code}

These are intentional and important:

\begin{enumerate}[leftmargin=*]
  \item The current reference is strictly 2D. The ROS code contains both 2D and 3D branches.
  \item Obstacles are currently circles. The ROS stack computes obstacle geometry from point clouds, polygonal segments, and local obstacle surfaces.
  \item The far-field goal controller is structurally inspired by the ROS code, but it is not yet a line-by-line reproduction of the SO(3)-style geometric block.
  \item The magnetic term is mathematically faithful in structure, but the current implementation uses a compact 2D analogue rather than a direct symbolic port of the original 3D cross-product expressions.
  \item The cleanroom model removes all ROS-specific concerns: TF, publishers/subscribers, launch files, point-cloud nodes, timestamps, and platform adapters.
\end{enumerate}

\subsection*{Why This Change Was Made}

The goal of this first cleanroom step is verification by isolation:

\begin{itemize}[leftmargin=*]
  \item keep the algorithm small enough to reason about
  \item remove ROS coupling
  \item make rollout behavior inspectable with CSV and plot outputs
  \item establish a stable place for future refactors and comparisons
\end{itemize}

\subsection*{Required Follow-Up}

Future major changes should update this file, especially if they modify:

\begin{itemize}[leftmargin=*]
  \item the exact goal-relaxation gain
  \item the magnetic obstacle formula
  \item the near-goal switching logic
  \item the state update model
  \item the obstacle abstraction
  \item the benchmark scenario assumptions
\end{itemize}

\section*{Entry 2026-05-22: Structural Refactor Toward The 2022 Paper}

\textbf{Status}

\begin{itemize}[leftmargin=*]
  \item Refactored in \texttt{src/mfinav/double\_integrator.py}
  \item This is a structural alignment step, not yet a full equation-level reconstruction of the 2022 paper
\end{itemize}

\textbf{Motivation}

The 2022 paper describes obstacle avoidance as the combination of two terms:

\begin{equation}
F_o = F_b + F_a
\end{equation}

where:

\begin{itemize}[leftmargin=*]
  \item $F_b$ is the boundary-following field
  \item $F_a$ is the collision-avoidance field
\end{itemize}

The earlier cleanroom prototype merged these ideas into one obstacle term plus
an added repulsive correction. That made the code easy to test, but it hid the
paper's core decomposition.

\textbf{What Changed}

The obstacle logic is now split into explicit components:

\begin{itemize}[leftmargin=*]
  \item \texttt{LocalSensingModel}
  \item \texttt{BoundaryFollowingField}
  \item \texttt{CollisionAvoidanceField}
\end{itemize}

The total control now has the explicit structure

\begin{equation}
a = a_{\text{goal}} + a_b + a_a
\end{equation}

where:

\begin{itemize}[leftmargin=*]
  \item $a_b$ is the boundary-following component
  \item $a_a$ is the collision-avoidance component
\end{itemize}

\textbf{Observation Model}

The refactor introduces a local observation container:

\begin{equation}
\mathcal{O} = \{ r_o,\; d_o \}
\end{equation}

where:

\begin{itemize}[leftmargin=*]
  \item $r_o$ is the closest-obstacle vector
  \item $d_o = \lVert r_o \rVert$
\end{itemize}

This is still generated from an analytic circle model in the current code, but
the new interface is intentionally shaped like a local sensing output so it can
be replaced later by sensor-driven obstacle processing.

\textbf{Boundary-Following Field}

The boundary-following component keeps the same current prototype behavior, but
it is now isolated as its own field:

\begin{equation}
\hat{t} = \hat{l}_a - (\hat{l}_a^\top \hat{r})\hat{r}
\end{equation}

\begin{equation}
B \propto \frac{\lVert v \rVert}{d}\hat{t}_{\perp}
\end{equation}

\begin{equation}
a_b = k_m \hat{l}_{a,\perp}(\hat{l}_{a,\perp}^\top B)
\end{equation}

This is still the compact 2D analogue used previously, but it is now clearly
identified as the boundary-following term.

\textbf{Collision-Avoidance Field}

The repulsive near-obstacle correction is now made explicit as the
collision-avoidance field:

\begin{equation}
a_a =
-k_r \left(\frac{1}{d} - \frac{1}{d_b}\right)\frac{r_o}{d}
\quad \text{for } d \le d_b
\end{equation}

and, when motion direction is defined, it is projected to remove the component
parallel to the current motion:

\begin{equation}
a_a^{\perp} = a_a - (a_a^\top \hat{l}_a)\hat{l}_a
\end{equation}

\textbf{Goal Relaxation Status}

The goal-relaxation logic remains available, but it is now explicitly marked as
a legacy extension from the ROS code rather than a confirmed core part of the
2022 paper implementation.

This is represented by a configuration switch:

\begin{equation}
\texttt{use\_legacy\_goal\_relaxation} \in \{0,1\}
\end{equation}

When disabled, the goal controller uses unit scaling rather than the legacy
relaxation factor.

\textbf{What This Refactor Improves}

\begin{itemize}[leftmargin=*]
  \item The code structure now mirrors the paper's decomposition more clearly.
  \item It becomes easier to replace the current placeholder formulas with
  paper-faithful ones.
  \item It isolates local sensing, boundary following, and collision avoidance
  as separate concerns.
\end{itemize}

\textbf{What Still Does Not Match The 2022 Paper Exactly}

\begin{itemize}[leftmargin=*]
  \item The exact paper equations for $F_b$ and $F_a$ are not yet reconstructed line by line.
  \item The threshold semantics of $r_l$ and $r_{la}$ are not yet implemented as separate paper-faithful activations.
  \item The Section 4.4 closest-point averaging extension is still missing.
  \item The current observation model still comes from an analytic obstacle, not real local sensing.
\end{itemize}

\section*{Entry 2026-05-22: Benchmark Environments and APF Baseline}

\textbf{Status}

\begin{itemize}[leftmargin=*]
  \item Added multi-obstacle environment support
  \item Added convex and non-convex benchmark scenarios
  \item Added a standard artificial potential field (APF) baseline
  \item Added a benchmark runner that saves trajectory plots and metrics
\end{itemize}

\textbf{What Changed}

The simulator is no longer restricted to a single obstacle. A new obstacle
collection abstraction returns the closest obstacle vector among several
primitive obstacles:

\begin{equation}
r_o = \arg\min_{r_i \in \mathcal{R}} \lVert r_i \rVert
\end{equation}

where $\mathcal{R}$ is the set of closest-point vectors returned by all
obstacles in the environment.

This enables more realistic environments while preserving the local-sensing
structure used by the controller.

To make controller comparisons numerically meaningful, the simulator now clips
both commanded acceleration and robot speed:

\begin{equation}
a \leftarrow
\begin{cases}
a, & \lVert a \rVert \le a_{\max} \\
a_{\max}\dfrac{a}{\lVert a \rVert}, & \text{otherwise}
\end{cases}
\end{equation}

\begin{equation}
v \leftarrow
\begin{cases}
v, & \lVert v \rVert \le v_{\max} \\
v_{\max}\dfrac{v}{\lVert v \rVert}, & \text{otherwise}
\end{cases}
\end{equation}

The rollout also terminates on collision, approximated by zero obstacle
clearance in the simplified geometric model.

\textbf{New Scenario Types}

Three benchmark scenarios are now defined:

\begin{itemize}[leftmargin=*]
  \item a single convex obstacle
  \item a cluster of convex obstacles
  \item a non-convex U-shaped obstacle approximated by several circles
\end{itemize}

The non-convex environment is still a geometric approximation, but it is useful
for testing whether the controller exhibits boundary-following behavior beyond
simple single-obstacle deflection.

\textbf{APF Baseline}

A standard artificial potential field baseline is added with the structure

\begin{equation}
a_{\text{APF}} = a_{\text{att}} + a_{\text{rep}}
\end{equation}

where the attractive term is

\begin{equation}
a_{\text{att}} = K_P (p_g - p) - K_D v
\end{equation}

and the repulsive term is activated near obstacles:

\begin{equation}
a_{\text{rep}}
=
-K_R \left(\frac{1}{d} - \frac{1}{d_b}\right)\frac{r_o}{d^2}
\quad \text{for } d \le d_b
\end{equation}

This baseline is not meant to reproduce every APF variant in the literature; it
is meant to provide a simple and interpretable comparison against the current
MFI implementation.

\textbf{Verification Outputs}

The benchmark runner now produces:

\begin{itemize}[leftmargin=*]
  \item trajectory comparison plots
  \item CSV metrics for each scenario and method
\end{itemize}

The current summary metrics are:

\begin{itemize}[leftmargin=*]
  \item success flag
  \item collision flag
  \item number of steps
  \item path length
  \item final goal distance
  \item minimum obstacle clearance
  \item mean speed
\end{itemize}

\textbf{Why This Matters}

This change gives us a repeatable way to evaluate:

\begin{itemize}[leftmargin=*]
  \item whether the current MFI implementation handles more complex geometry
  \item whether it offers any practical benefit over a standard APF baseline
  \item where the current simplified implementation breaks down
\end{itemize}

\section*{Entry 2026-05-22: Paper-Driven Thresholds, Averaging Extension, and Legacy Goal Relaxation Demotion}

\textbf{Status}

\begin{itemize}[leftmargin=*]
  \item Added explicit threshold semantics for boundary following and collision avoidance
  \item Added the Section 4.4 style closest-point averaging extension
  \item Changed legacy goal relaxation from default-on to optional
\end{itemize}

\textbf{Thresholded Obstacle Activation}

The implementation now follows the paper structure more closely by separating
the activation thresholds:

\begin{itemize}[leftmargin=*]
  \item boundary-following field active for $d \le r_l$
  \item collision-avoidance field active for $d \le r_{la}$
\end{itemize}

with default values:

\begin{equation}
r_l = 4.0, \qquad r_{la} = 2.5
\end{equation}

This replaces the previous use of generic proximity constants that did not map
cleanly to the paper's notation.

The implementation now also applies smooth activation weights:

\begin{equation}
\sigma_b =
\mathrm{clip}\left(
\frac{r_l - d}{r_l - r_{la}}, 0, 1
\right)
\end{equation}

\begin{equation}
\sigma_a =
\mathrm{clip}\left(
\frac{r_{la} - d}{r_{la}}, 0, 1
\right)
\end{equation}

so that the effective fields are:

\begin{equation}
a_b \leftarrow \sigma_b a_b
\end{equation}

\begin{equation}
a_a \leftarrow \sigma_a a_a
\end{equation}

This makes the obstacle interaction less abrupt and improved the clustered
obstacle benchmark in practice.

\textbf{Closest-Point Averaging Extension}

To approximate the Section 4.4 extension for non-unique closest points, the
observation model now collects all sensed obstacle vectors whose magnitudes are
within a threshold $\delta_r$ of the minimum sensed distance:

\begin{equation}
\mathcal{R}_{\delta}
=
\{ r_i \mid \lVert r_i \rVert \le r_{\min} + \delta_r \}
\end{equation}

The averaged vector is then

\begin{equation}
\bar{r}
=
\frac{1}{|\mathcal{R}_{\delta}|}
\sum_{r_i \in \mathcal{R}_{\delta}} r_i
\end{equation}

Following the paper's heuristic:

\begin{itemize}[leftmargin=*]
  \item if $\lVert \bar{r} \rVert < \lVert r_o \rVert$, use $\bar{r}$ as the working closest-point vector
  \item otherwise, use the standard closest vector $r_o$
\end{itemize}

This gives the code a practical non-unique closest-point handling mechanism for
concave and sharp-corner approximations.

To better reflect the paper's sensing assumption, the implementation no longer
uses only one exact closest-point vector per obstacle. Each obstacle now
provides multiple local surface samples around the nearest visible region, and
the averaging extension operates over these sensed vectors rather than over a
single analytic point per obstacle.

\textbf{Small-Current Fallback}

The paper notes that the projected artificial current can become very small when
the robot direction is nearly aligned with the robot-to-obstacle vector. To
mitigate this, the implementation now checks the magnitude of the projected
current and falls back to a unit tangent surrogate when:

\begin{equation}
\lVert \hat{t} \rVert < \epsilon
\end{equation}

with default

\begin{equation}
\epsilon = 3 \times 10^{-6}
\end{equation}

\textbf{Legacy Goal Relaxation}

The ROS-derived goal-relaxation term is no longer the default path for the
implementation. It is now explicitly treated as an optional legacy extension:

\begin{equation}
\texttt{use\_legacy\_goal\_relaxation} = 0
\end{equation}

by default.

When enabled, the implementation restores the earlier ROS-inspired scaling
factor. When disabled, the goal controller operates without that legacy gain
modulation.

\textbf{Why This Change Was Made}

These changes address three of the most important mismatches between the
prototype and the 2022 paper:

\begin{itemize}[leftmargin=*]
  \item explicit $r_l$ and $r_{la}$ threshold behavior
  \item handling of non-unique closest points
  \item clearer separation between paper-driven behavior and ROS-legacy behavior
\end{itemize}

\section*{Entry 2026-05-22: Tuned Defaults and Stronger Evaluation Metrics}

\textbf{Status}

\begin{itemize}[leftmargin=*]
  \item Increased the default rollout horizon from 2000 to 6000 steps
  \item Tuned the default interaction and sensing parameters around the better-performing region
  \item Added time-to-goal and path-efficiency evaluation metrics
\end{itemize}

\textbf{Tuned Defaults}

The default parameters are now:

\begin{equation}
r_l = 4.0, \quad r_{la} = 2.5, \quad c_{\text{field}} = 12.0, \quad c_{\perp} = 35.0
\end{equation}

and the local sensing surrogate uses:

\begin{equation}
N_{\text{samples}} = 15, \qquad \theta_{\text{window}} = 1.0, \qquad \delta_r = 0.5
\end{equation}

These values were selected because they materially improved the convex-cluster
benchmark while preserving good behavior in the single-convex case.

With the longer rollout horizon, the current MFI implementation is able to
reach the success region in all three default benchmark scenarios.

\textbf{Additional Evaluation Metrics}

The benchmark harness now reports:

\begin{itemize}[leftmargin=*]
  \item time-to-goal in simulation steps
  \item path efficiency
\end{itemize}

with

\begin{equation}
\text{path efficiency}
=
\frac{\lVert p_f - p_0 \rVert}{L}
\end{equation}

where:

\begin{itemize}[leftmargin=*]
  \item $p_0$ is the initial position
  \item $p_f$ is the realized final position
  \item $L$ is the realized trajectory length
\end{itemize}

This definition ensures the metric remains meaningful even when a method does
not reach the goal.

Time-to-goal is defined as the first step index where:

\begin{equation}
\lVert p - p_g \rVert \le r_{\text{success}}
\end{equation}

If the goal region is never reached, the value is reported as $\infty$.

\section*{Entry 2026-05-22: Polygonal Obstacles With Local Sensing}

\textbf{Status}

\begin{itemize}[leftmargin=*]
  \item Added explicit polygon obstacle support
  \item Kept the same local-sensing and closest-point averaging pipeline
  \item Converted the default benchmark environments to polygonal convex and non-convex obstacles
\end{itemize}

\textbf{Polygon Obstacle Model}

A polygon obstacle is represented by an ordered list of boundary vertices

\begin{equation}
\mathcal{P} = \{ v_1, v_2, \dots, v_N \}
\end{equation}

with edges formed by consecutive vertex pairs. For a robot position $p$, the
closest-point vector is computed by projecting onto each boundary segment and
selecting the shortest resulting vector.

\begin{equation}
r_o = \arg\min_{r_i \in \mathcal{E}} \lVert r_i \rVert
\end{equation}

where $\mathcal{E}$ is the set of candidate vectors to all polygon edges.

\textbf{Local Sensing on Polygon Boundaries}

Instead of using only a single exact closest point, the polygon model generates
multiple local samples along each edge around the nearest visible region. These
sampled boundary vectors are passed into the same observation model already used
for circular obstacles:

\begin{itemize}[leftmargin=*]
  \item find the minimum sensed distance
  \item gather all vectors within the averaging band $\delta_r$
  \item compute the averaged closest-point surrogate
  \item use the averaged vector when it is shorter than the single closest vector
\end{itemize}

This keeps the paper-style local sensing and closest-point averaging intact
while making convex and non-convex geometry easier to define.

\textbf{Why This Matters}

Polygonal obstacles let us represent:

\begin{itemize}[leftmargin=*]
  \item single convex obstacles such as boxes or rotated quadrilaterals
  \item clustered convex environments
  \item explicitly non-convex obstacles such as U-shaped boundaries
\end{itemize}

without approximating every case by a large collection of circles.

\section*{Entry 2026-05-22: Signed Clearance Evaluation and Head-On Boundary Fix}

\textbf{Status}

\begin{itemize}[leftmargin=*]
  \item Replaced unsigned polygon distance checks with signed obstacle clearance
  \item Added explicit safety-violation reporting in addition to hard collision reporting
  \item Changed the polygon boundary-following and repulsion terms so a head-on wall encounter no longer degenerates to zero avoidance command
\end{itemize}

\textbf{Signed Clearance}

For evaluation we now distinguish between:

\begin{itemize}[leftmargin=*]
  \item \emph{distance to the sensed boundary} used by the field law
  \item \emph{signed clearance} used by the evaluator
\end{itemize}

For circles,

\begin{equation}
c(p) = \lVert p - p_c \rVert - r
\end{equation}

For polygons, the evaluator computes the minimum Euclidean distance from the
robot position $p$ to any boundary edge and assigns a negative sign if $p$ lies
inside the polygon:

\begin{equation}
c(p) =
\begin{cases}
- \min\limits_{e \in \partial \mathcal{P}} d(p, e), & p \in \mathcal{P} \\
\min\limits_{e \in \partial \mathcal{P}} d(p, e), & p \notin \mathcal{P}
\end{cases}
\end{equation}

This means:

\begin{itemize}[leftmargin=*]
  \item $c(p) \le 0$ indicates collision / penetration
  \item $0 < c(p) \le c_{\text{safety}}$ indicates an unsafe low-clearance path
\end{itemize}

\textbf{Updated Success Accounting}

The benchmark now reports two success-style quantities:

\begin{equation}
\text{goal\_reached\_once} =
\begin{cases}
1, & \exists k \text{ such that } \lVert p_k - p_g \rVert \le r_{\text{success}} \text{ and } c_{\min} > c_{\text{safety}} \\
0, & \text{otherwise}
\end{cases}
\end{equation}

\begin{equation}
\text{success} =
\begin{cases}
1, & \lVert p_f - p_g \rVert \le r_{\text{success}} \text{ and } c_{\min} > c_{\text{safety}} \\
0, & \text{otherwise}
\end{cases}
\end{equation}

The first quantity captures whether the trajectory ever reached the goal region,
while the second remains the stricter final-state metric.

\textbf{Why The Old Polygon Case Failed}

The earlier 2D boundary-following and repulsion approximation had a degeneracy
for a head-on encounter with a vertical wall:

\begin{itemize}[leftmargin=*]
  \item the tangential term collapsed to zero because the projected field and the projected motion-perpendicular direction became orthogonal
  \item the repulsive term collapsed to zero because the full normal repulsion was projected away along the current motion direction
\end{itemize}

That is why parameter sweeps alone could not fix the U-shaped polygon case.

\textbf{Updated Field Form}

To remove that degeneracy, the boundary-following term now directly uses the
local tangent estimate:

\begin{equation}
a_b = \sigma_b \, c_{\text{field}} \, v_{\text{guide}} \, \frac{\hat{t}}{d}
\end{equation}

where

\begin{equation}
v_{\text{guide}} = \max(\lVert v \rVert, 0.5\,v_{\max})
\end{equation}

and $\hat{t}$ is the normalized local tangent estimate (or a perpendicular
fallback when the projected tangent becomes degenerate).

The collision-avoidance term now keeps its full normal component:

\begin{equation}
a_a =
- \sigma_a \, c_{\perp}
\left(
\frac{1}{d} - \frac{1}{r_{la}}
\right)
\frac{r_o}{d}
\end{equation}

This change is less ``projected'' than the previous 2D approximation, but it is
much more appropriate for polygonal wall encounters because it preserves a real
push-away component.

\section*{Entry 2026-06-18: Closer Reconstruction of Paper PD, Geometric, \(F_b\), and \(F_a\)}

\textbf{Status}

\begin{itemize}[leftmargin=*]
  \item Added separate paper-style PD and paper-style geometric presets
  \item Replaced the earlier paper placeholder \(F_b\) with a closer cross-product-based reconstruction
  \item Replaced the earlier smoothed paper placeholder \(F_a\) with a hard-threshold projected repulsive term
  \item Added memory of the previously proposed obstacle-surface current for the small-current degeneracy case
\end{itemize}

\textbf{Paper-style Goal Modes}

The repository now distinguishes:

\begin{itemize}[leftmargin=*]
  \item \texttt{paper\_pd}: direct attractive PD goal control
  \item \texttt{paper\_geometric}: speed regulation plus steering correction
  \item \texttt{pragmatic}: the cleaner benchmark-oriented hybrid controller used in earlier refactors
\end{itemize}

For \texttt{paper\_pd}, the implemented goal term is:

\begin{equation}
u_g = K_P (p_g - p) - K_D \dot{p}
\end{equation}

For \texttt{paper\_geometric}, the far-field speed regulation remains:

\begin{equation}
u_{\text{attr}} = K_{P,r} (v_{\max} - \lVert v \rVert)\hat{v}
\end{equation}

and the steering term is now treated as a 2D analogue of the old SO(3)-based
rotation controller:

\begin{equation}
u_{\text{steer}} = \omega \, v_{\perp},
\qquad
\omega = K_{\omega}\,\mathrm{angle}(v, p_g-p)
\end{equation}

\textbf{Paper-style Boundary-Following Field}

The previous ``paper mode'' used a smoothed 2D surrogate. The updated
implementation is closer to the old ROS magnetic construction and to the paper
derivation:

\begin{equation}
l_o = l_a - \left(l_a^\top \hat{r}\right)\hat{r}
\end{equation}

\begin{equation}
B = [l_o]_{\times} v / d
\end{equation}

\begin{equation}
F_b = c [l_a]_{\times} B
\end{equation}

The implementation now uses a hard activation at \(d \le r_l\) for paper mode
instead of the earlier smooth activation weight.

\textbf{Small-current degeneracy handling}

When \(\lVert l_o \rVert\) becomes too small, the implementation now reuses the
previously proposed normalized obstacle-surface current instead of immediately
switching to the pragmatic tangent fallback. This follows the paper discussion
more closely than the earlier heuristic.

\textbf{Paper-style Collision-Avoidance Field}

For paper mode, the implementation now applies:

\begin{equation}
F_a =
- c_{\perp}
\left(
\frac{1}{d} - \frac{1}{r_{la}}
\right)
\frac{r_o}{d}
\end{equation}

with a projection that removes the component along the instantaneous robot
motion direction in order to better preserve the paper property that \(F_a\)
does not change speed.

\textbf{Observed outcome}

These changes made the paper-aligned PD controller much more plausible on the
simple and clustered convex polygon scenarios. The paper-aligned geometric mode
also became stable after correcting the steering sign in the 2D analogue.

However, the paper-aligned modes still collide in the non-convex U-shaped
polygon benchmark under the current local polygon sensing abstraction, so the
equation-level reconstruction is improved but still not complete.
