Paper 2022 Metric Summary

Friendlier benchmark summary across all current models and navigators. Green marks the best value and red marks the worst value within each table block.

Common benchmark step size: dt = 0.02 s.

Metric Glossary

  • Succ: number of successful runs in that table block.
  • Succ Rate: fraction of runs that finish inside the goal tolerance while also satisfying the safety criterion.
  • Reach Once: fraction of runs that enter the goal region at least once, even if they do not stay there at the final state.
  • Path: traveled trajectory length.
  • T_goal (s): time to first reach the goal region, averaged only over runs with finite goal-reaching time.
  • Final Err: final distance to the goal, used here as an accuracy proxy.
  • Mean Clr: average closest-obstacle clearance summary for the block.
  • Worst Clr: smallest clearance observed in the block. Negative means penetration or collision.
  • Coll Rate: fraction of runs whose minimum clearance goes below zero.
  • Safe Viol: fraction of runs whose minimum clearance drops below the configured safety clearance margin.
  • Eff: path efficiency, computed as straight-line displacement divided by traveled path length.

How To Read Colors

  • Green marks the best value within the current table block.
  • Red marks the worst value within the current table block.
  • Aggregate tables compare methods within the same model family.
  • Per-scenario tables compare methods within the same scenario.
  • For success, clearance, and efficiency, larger is better.
  • For time, final error, collision rate, and safety violation rate, smaller is better.
  • Cells with - mean the metric is undefined for that row, usually because the method never reached the goal.

Overall Summary

Across all current benchmarks, the strongest method in general is haddadin, with the highest average success rate (0.700) across the model families now implemented.

The weakest method in general is sabattini_3d, with the lowest average success rate (0.071).

The most reliable "almost-there" behavior comes from paper_geometric_3d, which has the best average goal-region reach-once rate (0.785). The best average final accuracy comes from paper_pd_3d with mean final error 1.006.

From a safety perspective, the best average collision rate is achieved by apf (0.000), while the worst average collision rate belongs to sabattini (0.800).

At the model-specific level, the single strongest benchmark block right now is double_integrator_3d / paper_pd_3d with success rate 1.000. The weakest block is quadrotor_3d / apf_3d with success rate 0.000.

A comparison caveat is important here: the current Haddadin baseline is not purely local in the same sense as MFI. In this clean repo it uses local obstacle distance and direction, but it also uses obstacle center or centroid information to build its transverse avoidance term. MFI, by contrast, relies only on local obstacle sensing, closest-point averaging, and goal relaxation. So when MFI performs comparably well, or better in some cases, it is doing so under a weaker information assumption and with a more reactive formulation.

In practical terms, the current repo is strongest for the point-mass double-integrator benchmarks, especially the 3D paper-style variants. The main remaining weakness is the quadrotor benchmark family, where all methods still lag behind the point-mass results and need more controller tuning or model-specific navigation adaptation.
